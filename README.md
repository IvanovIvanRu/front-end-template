# 🚀 Шаблон простейшей фронтенд-сборки на Vite

Добро пожаловать на практическое занятие! Этот репозиторий представляет собой готовую минималистичную среду разработки (сборку) для работы с чистым веб-стеком: **HTML, CSS, JS и графическими ассетами**.

Сборка построена на базе **Vite** — современного, супербыстрого инструмента, который избавляет разработчика от рутины и автоматизирует ключевые процессы.

---

## 🛠 Что под капотом (Что и зачем установилось)

После развёртывания проекта и выполнения команды `npm install` в папку `node_modules` устанавливаются следующие зависимости:

1.  **`vite` (Vite)** — наш главный инструмент (сборщик и локальный сервер).

- _Зачем на этапе разработки:_ Он запускает локальный веб-сервер и включает **HMR (Hot Module Replacement)**. Это значит, что при редактировании и сохранении HTML, CSS или JS кода страница в браузере мгновенно обновляется сама — нажимать «Обновить» вручную больше не нужно.

- _Зачем на этапе сборки:_ Он оптимизирует код для продакшена и автоматически добавляет уникальные **хэш-метки в имена файлов** (например, `main-C4b8Xz1a.js`). Это предотвращает кеширование: при обновлении сайта на хостинге у пользователей гарантированно загрузится самая свежая версия.

2.  **`rolldown` / `esbuild**` — внутренние движки Vite, которые отвечают за моментальную компиляцию кода, связывание ES-модулей (`type="module"`) и сборку картинок в итоговый бандл.

## 📂 Структура проекта

Для удобства ваш рабочий код изолирован в папке `src`, а настройки и точка входа находятся в корне:

- `index.html` — главная страница сайта (точка входа для Vite).

- `vite.config.js` — конфигурационный файл, который управляет правилами сборки и распределением файлов по папкам.

- `src/` — папка со всеми исходниками (исходным кодом):

- `src/assets/` — сюда складываются картинки (PNG, JPG, SVG и др.).

- `src/css/style.css` — ваши стили.

- `src/js/main.js` — логика на JavaScript.

## 💻 Инструкция по использованию

### 1. Подготовка (выполняется один раз при старте)

Убедитесь, что на вашем компьютере установлена **Node.js**. Откройте терминал в папке проекта и установите все необходимые зависимости командой:

```bash
npm install

```

### 2. Режим разработки (Локальный сервер + авторелоад)

Чтобы запустить локальный сервер для написания кода, выполните в терминале:

```bash
npm run dev

```

- В терминале появится ссылка (обычно `http://localhost:5173/`). Откройте её в браузере.

- Меняйте код в файлах `index.html`, `style.css` или `main.js`, нажимайте `Ctrl+S` (`Cmd+S`), и изменения тут же появятся на экране!

### 3. Финальная сборка (Для выкатки на хостинг)

Когда ваш сайт будет полностью готов к публикации в интернете, остановите сервер и выполните команду:

```bash
npm run build

```

- Vite создаст в корне проекта новую папку **`dist`**.

- Внутри неё будет лежать чистый, сжатый и оптимизированный код, а все ассеты (скрипты, стили, изображения) получат уникальные хэш-метки от кеширования и распределятся по своим папкам.

- Чтобы опубликовать сайт, достаточно просто скопировать содержимое папки `dist` на ваш хостинг.

## 💡 Важная памятка по импорту картинок

Благодаря конфигурации в этой сборке, вы можете подключать изображения двумя способами:

1.  **Через HTML:** Прямо в теге `<img>` указывать относительный путь: `<img src="./src/assets/logo.png">`.

2.  **Через JavaScript (Возможность Vite):** Вы можете импортировать файл картинки как модуль прямо в JS-код, и Vite сам подставит правильный путь после сборки:

```javascript
import logoUrl from "../assets/logo.png";
document.getElementById("my-img").src = logoUrl;
```
